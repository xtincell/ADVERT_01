## Mode d'emploi

Ce protocole guide la collecte, le stockage et l'utilisation éthique des données marché dans le cadre du Pilier T (Track). Il s'applique à toutes les données utilisées pour valider les hypothèses stratégiques des Piliers A→E.

**Objectif :** Assurer que chaque affirmation dans le document de stratégie 360° (Pilier S) repose sur des données vérifiables, croisées et éthiquement collectées.

**Principes directeurs :**
1. **Triangulation** — Chaque affirmation clé doit être corroborée par au moins 2 sources indépendantes
2. **Traçabilité** — Chaque donnée doit être reliée à sa source
3. **Fraîcheur** — Les données de plus de 18 mois doivent être revalidées
4. **Éthique** — Conformité aux lois de protection des données du marché cible (cf. Annexe F)
5. **Transparence** — Le client doit connaître les sources et leur niveau de confiance

## Étape 1 : Sources de Données

### Catégorisation des Sources

Le Pilier T croise **3 types de sources** pour valider les hypothèses. Chaque source a un niveau de confiance et une méthode de collecte propre.

#### Source Type 1 — Données Existantes (Internes)

**Définition :** Données que le client possède DÉJÀ — CRM, analytics, rapports, bilans.

| Source | Où la trouver | Confiance | Effort de collecte |
|---|---|---|---|
| CRM (Salesforce, HubSpot, etc.) | Service commercial | Élevée | Faible — export CSV |
| Google Analytics / équivalent | Service marketing / DSI | Élevée | Faible — export |
| Rapports de vente (12 derniers mois) | Direction financière | Élevée | Faible |
| Études de satisfaction client | Service qualité | Moyenne-Élevée | Faible |
| Données de rétention / churn | CRM ou service client | Élevée | Moyen |
| Bilans financiers | Direction financière | Élevée | Moyen |
| Historique des campagnes | Service marketing | Moyenne | Faible |
| Feedback interne (NPS interne) | RH | Moyenne | Moyen |

**Checklist de collecte :**

| # | Action | Statut |
|---|---|---|
| 1 | ☐ Demander l'accès aux données AVANT le premier atelier | |
| 2 | ☐ Signer un NDA si les données sont sensibles | |
| 3 | ☐ Identifier un point de contact data chez le client | |
| 4 | ☐ Vérifier la qualité des données (complétude, fraîcheur, cohérence) | |
| 5 | ☐ Documenter les limitations (données manquantes, biais connus) | |

#### Source Type 2 — Données en Ligne (Externes)

**Définition :** Données publiquement disponibles — études de marché, rapports sectoriels, données statistiques, veille concurrentielle.

| Source | Exemples | Confiance | Effort de collecte |
|---|---|---|---|
| Rapports d'analystes | McKinsey, Deloitte, PwC Africa | Élevée | Moyen — souvent payants |
| Bases de données statistiques | Banque Mondiale, AfDB, Statista | Élevée | Faible — accès en ligne |
| Rapports sectoriels locaux | Chambres de commerce, ministères | Moyenne-Élevée | Moyen |
| Données concurrentielles publiques | Sites web, réseaux sociaux, Crunchbase | Moyenne | Moyen |
| Avis et commentaires clients | Google Reviews, Trustpilot, réseaux sociaux | Moyenne | Moyen |
| Publications académiques | Google Scholar, ResearchGate | Élevée | Moyen |
| Articles de presse spécialisée | TechCabal, Jeune Afrique, Disrupt Africa | Moyenne | Faible |
| Données de trafic web | SimilarWeb, SEMrush | Moyenne | Faible-Moyen |

**Checklist de collecte :**

| # | Action | Statut |
|---|---|---|
| 1 | ☐ Prioriser les sources à haute confiance (rapports d'analystes, bases statistiques) | |
| 2 | ☐ Vérifier la date de publication (< 18 mois) | |
| 3 | ☐ Documenter l'URL ou la référence complète | |
| 4 | ☐ Croiser avec au moins une source indépendante | |
| 5 | ☐ Signaler le niveau de confiance dans le document de stratégie | |

#### Source Type 3 — Entretiens (Terrain)

**Définition :** Données collectées directement via les entretiens avec le fondateur, l'équipe, les clients, les partenaires.

| Type d'entretien | Cible | Confiance | Effort |
|---|---|---|---|
| Entretien fondateur | Fondateur / CEO | Élevée (pour vision), Moyenne (pour données) | Élevé — 2-3h |
| Entretien équipe dirigeante | Directeurs (CMO, CFO, CTO) | Élevée | Moyen — 1h chacun |
| Entretien client fidèle | Top 5-10 clients | Élevée | Moyen — 30-45min chacun |
| Entretien client perdu | Ex-clients | Très Élevée (pour insights négatifs) | Élevé — difficile à obtenir |
| Entretien partenaire | Distributeurs, fournisseurs | Moyenne | Moyen — 30min |
| Entretien expert sectoriel | Journaliste, analyste, universitaire | Élevée | Élevé — accès difficile |

**Checklist de collecte :**

| # | Action | Statut |
|---|---|---|
| 1 | ☐ Préparer le guide d'entretien (cf. Annexe C) | |
| 2 | ☐ Obtenir le consentement écrit pour enregistrement (si applicable) | |
| 3 | ☐ Informer l'interviewé de la finalité et de l'utilisation des données | |
| 4 | ☐ Prendre des notes en temps réel — ne PAS compter uniquement sur l'enregistrement | |
| 5 | ☐ Identifier les verbatims clés pour réutilisation dans le document | |
| 6 | ☐ Anonymiser les données sensibles dans le livrable final | |

## Étape 3 : Validation Croisée

### Règles de Triangulation

| Scénario | Verdict | Action |
|---|---|---|
| 3 sources confirment | **Validée** — Haute confiance | Intégrer dans le document 360° |
| 2 confirment, 1 contredit | **Validée avec nuance** — Moyenne confiance | Intégrer avec réserve documentée |
| 2 confirment, 1 absente | **Validée** — Moyenne confiance | Intégrer, signaler le gap |
| 1 confirme, 1 contredit | **Non conclusive** | Chercher une 3ème source |
| 1 seule source | **Non validée** | Ne PAS utiliser comme fait — qualifier d'hypothèse |
| 0 source | **Hypothèse non vérifiée** | Exclure ou indiquer clairement « hypothèse » |

## Étape 4 : Stockage et Archivage

### Structure de Dossier Recommandée

```
[NOM_CLIENT]_ADVERT/
├── 01_Données_Internes/
│   ├── CRM_export_[date].csv
│   ├── Analytics_[date].pdf
│   └── Rapports_financiers/
├── 02_Données_En_Ligne/
│   ├── Rapports_analystes/
│   ├── Veille_concurrentielle/
│   └── Sources_web/ (captures d'écran datées)
├── 03_Entretiens/
│   ├── Transcriptions/
│   ├── Notes_brutes/
│   └── Synthèses/
├── 04_Validation/
│   ├── Matrice_validation.xlsx
│   └── Fiches_collecte/
├── 05_Livrables/
│   ├── Document_strategie_360_v[X].pdf
│   └── Annexes/
└── README.md (index du dossier + dates + contact client)
```

### Règles de Stockage

| Règle | Détail |
|---|---|
| **Nommage** | `[TYPE]_[SOURCE]_[DATE].[ext]` — ex. : `ENTRETIEN_CEO_2026-03-15.docx` |
| **Versionnage** | Garder toutes les versions — ne jamais écraser un fichier |
| **Chiffrement** | Les données clients sensibles doivent être chiffrées (AES-256 minimum) |
| **Accès** | Dossier partagé uniquement avec l'équipe projet — pas de lien public |
| **Sauvegarde** | Minimum 2 copies : cloud sécurisé + local |
| **Durée de rétention** | 3 ans après la livraison du projet (sauf obligation légale différente) |
| **Suppression** | Données personnelles supprimées sur demande du client (cf. Annexe F) |

### Conformité Data par Marché

| Marché | Loi applicable | Exigence clé | Impact sur le stockage |
|---|---|---|---|
| Afrique du Sud | POPIA | Consentement + notification 72h | Hébergement sécurisé obligatoire |
| Nigeria | NDPA | Audit annuel si >200 sujets | DPO + registre de traitements |
| Kenya | DPA 2019 | DPIA pour traitements à risque | Évaluation d'impact préalable |
| UEMOA/CEMAC | Lois nationales | Déclaration auprès de l'autorité | Varie selon le pays |
| Maroc | Loi 09-08 | Autorisation CNDP pour transferts | Localisation possible |
| Tunisie | Loi 2004-63 | Déclaration INPDP | Consentement renforcé |

## Étape 5 : Mise à Jour Périodique

### Calendrier de Rafraîchissement

Le marché bouge. Les données vieillissent. Voici le calendrier recommandé pour maintenir la stratégie à jour.

| Type de donnée | Fréquence de mise à jour | Responsable | Déclencheur de mise à jour urgente |
|---|---|---|---|
| Données CRM / ventes | Mensuelle | Client | Chute >20% d'un KPI clé |
| Analytics web/social | Mensuelle | Client ou consultant | Changement d'algorithme plateforme |
| Veille concurrentielle | Trimestrielle | Consultant | Entrée d'un nouveau concurrent |
| Données de marché (TAM/SAM) | Semestrielle | Consultant | Choc économique / réglementaire |
| Satisfaction client (NPS) | Trimestrielle | Client | Crise de réputation |
| Données réglementaires | Semestrielle | Consultant + avocat | Nouvelle loi adoptée |
| Entretiens de validation | Annuelle | Consultant | Pivot stratégique du client |

### Signaux d'Alerte — Quand Tout Reprendre

Certains événements rendent une révision COMPLÈTE de la stratégie nécessaire :

| Signal | Piliers impactés | Action |
|---|---|---|
| Changement de CEO / fondateur | A (tous), D4, S | Reprendre le Pilier A à zéro |
| Entrée d'un concurrent majeur | D2, D4, R, T | SWOT complet + repositionnement |
| Changement réglementaire majeur | R, T, F (conformité) | Audit de conformité immédiat |
| Crise de réputation | A1, D3, D5, E3 | Plan de gestion de crise |
| Pivot produit/service | V (tous), D3, E | Reprendre le Pilier V à zéro |
| Fusion / acquisition | Tous les piliers | Reprise ADVERT complète |
| Effondrement d'un KPI critique (>50%) | E5, E6, V6 | Investigation + plan correctif immédiat |

---

**Rappel :** Les données sont le carburant du Pilier T et la fondation de crédibilité de tout le document 360°. Une donnée non sourcée est une opinion. Une opinion dans un document de stratégie est un risque. Sourcez tout. Croisez tout. Documentez tout.
